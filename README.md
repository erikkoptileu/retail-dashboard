# Retail Analytics Dashboard

Аналитический дашборд для RetailCRM с синхронизацией в Supabase, визуализацией продаж и Telegram-уведомлениями.

🔗 **Live:** https://retail-dashboard-theta.vercel.app

---

## О проекте

Проект сделан за **1.5–2 часа** в паре с Claude Code. Каждая задача решалась отдельно: сначала промпт → выполнение → проверка результата → следующий шаг.

## Что делали и какие промпты давали

### 1. Импорт заказов в RetailCRM
> *"В этой папке есть upload_orders.py скрипт для импорта и mock_orders.json — сейчас она выдаёт ошибку [50/50 ошибок]. Проверь пожалуйста"*

**Проблема:** `orderType: "eshop-individual"` не существовал в системе — только `"main"`.
**Решение:** Claude запросил справочники через API, нашёл доступные типы и исправил одну строку.

---

### 2. Импорт с правильными товарами
> *"Сейчас заказы импортировались без товаров. Можем сначала проверить есть ли товар в CRM, если нет — создать с наименованием и ценой, далее создать заказ?"*

**Проблема:** `productName` передавался внутри `offer.name` → отображался как "noname".
**Решение:** Перенёс `productName` на уровень item — RetailCRM сам создаёт позицию с нужным названием. Результат: 50/50 заказов загружены корректно.

---

### 3. Синхронизация RetailCRM → Supabase
> *"Теперь давай напишем скрипт который забирает заказы из RetailCRM API и кладёт в Supabase"*

Создан `sync_orders.py` с тремя таблицами: `customers`, `orders`, `order_items`. Upsert через Supabase REST API. Загружено: 51 клиент, 51 заказ, 86 позиций.

---

### 4. Дашборд визуализации
> *"График заказов и пару ключевых метрик для продаж. Нужно задеплоить на Vercel через GitHub"*

**Где застряли:**
- Chart.js падал с ошибкой `"category" is not a registered scale` — SSR Next.js пытался рендерить графики на сервере без доступа к `window`.
- **Решение:** Регистрация Chart.js перенесена в `useEffect`, компоненты загружаются через `dynamic import` с `ssr: false`.

---

### 5. Редизайн UI + новые страницы
> *"Сделай тему светлым, добавь адаптивку, сделай активными вкладки: Продажи, Товары и удали Клиенты"*

Добавлены страницы `/sales` и `/products` с общим `Layout` компонентом. Переключение через Next.js роутер с подсветкой активной вкладки.

---

### 6. Telegram-уведомления
> *"Настрой уведомление в Telegram когда в RetailCRM появляется заказ на сумму больше 50,000 ₸"*

**Как сделано:**
- В RetailCRM создан триггер **"Изменение заказа"** с условием `changeSet.isCreate and order.totalSumm > 50000`
- Действие: HTTP-запрос напрямую к Telegram Bot API с параметрами `chat_id` и `text` через шаблонные переменные `{{ order.firstName }}`, `{{ order.totalSumm }}` и т.д.

## Стек

| Слой | Технология |
|------|-----------|
| **CRM** | RetailCRM (API v5) |
| **База данных** | Supabase (PostgreSQL) |
| **Frontend** | Next.js 14, React 18 |
| **Графики** | Chart.js + react-chartjs-2 |
| **Стили** | CSS Modules |
| **Деплой** | Vercel (GitHub CI/CD) |
| **Уведомления** | Telegram Bot API + RetailCRM Триггеры |
| **Скрипты** | Python 3 (requests) |

---

## Архитектура

```
RetailCRM API
    │
    ├── upload_orders.py      → импорт заказов из JSON в CRM
    ├── sync_orders.py        → синхронизация CRM → Supabase
    │
    └── Триггер (>50k ₸)     → Telegram Bot API
             │
             └── уведомление в группу

Supabase (PostgreSQL)
    ├── customers
    ├── orders
    └── order_items
             │
             └── Next.js API routes → дашборд на Vercel
```
